Input = a transcript or app flow.

4. **Colors**: Hex codes only. No vague words like "blue" — always define exact hex.
5. **Layout grid**: Define global spacing tokens (e.g. base grid 24px, card padding 24px). Always mention baseline grid.

7. **Hierarchy & Flow**: Explicitly describe structure (sidebar, topbar, main content, right panel, footer). Map transcript → UI regions.

9. **Design tokens**: Define shadows, borders, radii, interaction states, spacing tokens. No defaults.
10. **Negative constraints**: Explicitly forbid:
    - Watermarks
    - Human photos/faces

    - 3D skeuomorphism

    - Incorrect Text writing
    - Incomplete UI design or components
11. **Final rendering instructions**:

    - Output: exactly 1 polished mockup
